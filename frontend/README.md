# Frontend

React 19 + TypeScript + Vite + Mantine + TanStack Query, with the typed API
client generated from the backend's OpenAPI schema.

See the [top-level README](../README.md) for the full dev workflow and the
backend env-var surface. This file is the frontend-specific view.

## Common commands

All run through `mise` from the repo root so the pinned Node (24.15.0) and
the `packageManager`-pinned pnpm are picked up automatically:

```sh
mise run dev:frontend         # vite :5173, proxies /api → :8000
mise run lint:frontend        # biome check (lint + format check in one pass)
mise run typecheck:frontend   # tsc -b --noEmit
mise run test:frontend        # vitest run
mise run fix:frontend         # biome check --write (auto-fix + format)
mise run build:frontend       # tsc -b && vite build → dist/
mise run generate:client      # regen src/api/* from the live backend
```

For one-offs, run `pnpm …` from this directory — the corepack shim picks
up the pinned pnpm version from `package.json`'s `packageManager` field.

| `pnpm` script        | Purpose                                          |
|----------------------|--------------------------------------------------|
| `pnpm dev`           | Vite dev server                                  |
| `pnpm build`         | Type-check + production bundle into `dist/`      |
| `pnpm generate`      | Regenerate `src/api/*` via `@hey-api/openapi-ts` |
| `pnpm test`          | `vitest run` (single pass)                       |
| `pnpm test:watch`    | `vitest` watch mode                              |
| `pnpm test:coverage` | `vitest run --coverage` (v8)                     |
| `pnpm test:e2e`      | Playwright suite in `e2e/`                       |
| `pnpm test:e2e:ui`   | Playwright UI mode                               |
| `pnpm storybook`     | Storybook dev server on :6006                    |
| `pnpm docs`          | TypeDoc → markdown reference                     |

## Layout

```
src/
  App.tsx           ← top-level router + layout
  main.tsx          ← React entrypoint + Mantine provider
  theme.ts          ← Mantine theme (amber/ink palette)
  api/              ← generated by @hey-api/openapi-ts — do NOT edit
  components/       ← one file per component, including *.stories.tsx
  hooks/            ← reusable query/mutation/notification hooks
  lib/              ← status helpers, polling, error formatting, …
  styles/           ← global CSS + design tokens
  test/             ← vitest setup + shared utilities
e2e/                ← Playwright specs (boots backend + frontend in CI)
.storybook/         ← Storybook config
designshots/        ← Playwright spec that captures design screenshots
public/             ← static assets served as-is (favicon, manifest, …)
```

`src/api/` is regenerated by `@hey-api/openapi-ts` from the running
backend's OpenAPI schema — never hand-edit it. Run `mise run generate:client`
(or `pnpm generate`) against a live backend after changing any
`backend/src/backend/*/router.py` or `*/schemas.py`.

## End-to-end tests

`e2e/` runs against a real backend that the workflow boots via
`backend/tests/e2e_server.py` (a thin shim that wires the FastAPI app to a
disposable data dir). `playwright.config.ts` reuses an existing server if
one is up, otherwise spawns both. In CI the servers are started ahead of
time so their logs land in the GitHub Actions log.

```sh
pnpm test:e2e               # headless run
pnpm test:e2e:ui            # interactive runner
pnpm exec playwright show-report   # open the last HTML report
```

## Storybook

`pnpm storybook` opens the component sandbox on
[http://localhost:6006](http://localhost:6006). Stories live next to their
component as `*.stories.tsx`.
